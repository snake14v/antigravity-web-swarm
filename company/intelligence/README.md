## WHAT'S IN THIS FOLDER

Four documents that form the complete business strategy framework:

### 1. **Product_Focus_Policy.md**
The official constraint document. Declares ShopSense as the single product in focus and defines the status of all other initiatives.

**Use when:**
- You want to start a new feature or product
- Someone suggests working on something else
- You need to justify why we're NOT doing something

**Key sections:**
- Policy statement (ShopSense only, everything else frozen/paused)
- Product status definitions (ACTIVE, FROZEN, PAUSED, DEFERRED, SUNSET)
- Current decisions for each product (ShopSense, OLOG Platform, Web Design, AurumGuard, Surveillance/IoT)
- Decision rules for new ideas (Rule 1-4 forcing questions)
- Gates and milestones (5 gates from first pilot to Series A)
- Exception process (how to break the policy when needed)

**Actions:**
- Review before starting new work
- Update PROGRESS.md if you break this policy
- Revisit on June 30 when Gate 3 is reached

### 2. **Hypothesis_Test_Framework.md**
The scientific validation plan. Tests whether the core assumption is true: "Bangalore chai shop owners will pay ₹25,000 for ShopSense."

**Use when:**
- Running pilot programs
- Deciding what to learn from each customer conversation
- Measuring success of the April-June test period
- Making pivot decisions

**Key sections:**
- The central hypothesis (clear, measurable, high-stakes)
- Why it matters (cascade of consequences if false)
- Experiment design (April 1 — June 30 test, 10 pilots, 30% conversion target)
- Variables (what we change, what we keep constant)
- Measurement protocol (track source, shop type, outcome for each pilot)
- Pivot triggers (what signals tell us to change strategy)
- Decision gate meeting template (June 30 review)

**Actions:**
- Update weekly with pilot data
- Flag any pivot triggers immediately
- Hold June 30 decision gate meeting
- Document outcomes in PROGRESS.md

- Competitive advantage checklist (verify we're still ahead)

**Actions:**
- Run monthly searches on specified keywords (last Friday of month)
- Update competitor profiles when news appears
- Complete monthly intel update template
- Assess threat level changes
- Trigger strategic response if any early warning signals activate

### 4. **Weekly_Business_Review.xlsx**
Professional Excel workbook for tracking metrics and progress.

**Use when:**
- Monday morning (fill in this week's data)
- Friday (review the week)
- Month-end (check if on track for gates)

**Sheets:**

**Sheet 1: Weekly Dashboard**
- Current week (auto-calculated via formula)
- 7 KPIs: Revenue, Units, Pilots, Paying Customers, LinkedIn, GitHub, Hypothesis On Track
- Weekly priority (text)
- Biggest blocker (text)
- Customer conversation this week (Y/N)

**Sheet 2: Weekly Log (52 weeks)**
- Rolling log of all 52 weeks from April 7, 2026 — April 3, 2027
- Track: Revenue, units, pilots, paying customers, priority, wins, blockers
- Use to spot trends (is revenue growing? Are we hitting pilot targets?)

**Sheet 3: Hypothesis Tracker**
- 10 pilot experiments
- Track: Shop name, type, contact date, start/end, outcome, decline reason, source
- Auto-calculates conversion rate
- Use to assess whether hypothesis is TRUE/FALSE by June 30

**Sheet 4: Competitive Watch**
- 5 competitors (SnapBizz, Khatabook, DotPe, OkCredit, New Entrants)
- Track: Last checked, key activity, threat level, next check date
- Use for monthly intelligence updates

**Actions:**
- Fill every Monday with this week's data
- Review before weekly planning meetings
- Use for monthly board/advisor updates

## HOW TO USE THESE DOCUMENTS TOGETHER

### FLOW 1: You Want to Build Something New
1. Read **Product_Focus_Policy.md** → Apply Decision Rules (Does it serve ShopSense in next 30 days?)
2. If no → Write to BACKLOG.md, do not start
3. If yes → Record in PROGRESS.md, proceed

### FLOW 2: Running Pilots (April-June 2026)
1. Every time you offer a pilot → Record in **Hypothesis_Test_Framework.md** pilot tracker
2. After pilot ends → Update Weekly_Business_Review.xlsx Sheet 3 (Hypothesis Tracker)
3. Every month → Update Sheet 2 (Weekly Log) with cumulative progress
4. June 30 → Hold Gate decision meeting using Hypothesis_Test_Framework template

### FLOW 3: Competitive Intel (Monthly)
1. Last Friday of each month → Complete searches from **Competitive_Intelligence_Tracker.md**
2. If anything found → Update competitor profile + threat level
3. If any early warning signal activated → Schedule emergency 48-hour response plan
4. Update **Weekly_Business_Review.xlsx** Sheet 4 with latest findings

### FLOW 4: Weekly Accountability
1. Monday morning → Update Weekly_Business_Review.xlsx with this week's data
2. Friday → Review the week, assess if on track for gates
3. Any blockers → Add to Weekly Dashboard + plan resolution
4. Any customer conversations → Log them (drives Hypothesis_Test_Framework)

## KEY METRICS TO TRACK

**Revenue** (from pilot conversions)
- April target: ₹0 (proving market, not revenue yet)
- May target: ₹25,000+ (first paid customer)
- June target: ₹75,000+ (3 paid customers)

**Units Deployed**
- = Number of ShopSense hardware units in field (pilots + paid)

**Active Pilots**
- = Number of free pilots running right now

**Paying Customers**
- The north star metric
- April: 0 (acceptable)
- May 31: 1+ (hypothesis still alive)
- June 30: 3+ (hypothesis proven, scale aggressively)
- June 30: 0 (hypothesis dead, prepare to pivot)

**Hypothesis On Track**
- Y if conversion rate trending toward 30%
- N if stuck at 0% by mid-June
