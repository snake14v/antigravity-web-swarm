5-Layer Company OS Architecture

Ooru Logix operates as an AI-native company built on five interconnected layers:

LAYER 1: AGENTS → 18 AI workers across 4 departments (Sales, Marketing, Engineering, Finance/Ops) LAYER 2: SKILLS → 28 executable capabilities agents invoke to complete tasks LAYER 3: KNOWLEDGE → Persistent memory that survives across sessions (Obsidian vault) LAYER 4: MCP SERVERS → 7 authenticated bridges to external platforms LAYER 5: PLUGINS → 5 bundled department packages (agents + skills + MCP)

Design Principles

Agents are stateless

They read knowledge, act, and write back. No hidden state between sessions.

Skills are pure functions

Given inputs, they produce outputs. Side effects only via MCP servers.

Knowledge is the single source of truth

If it's not in the vault, it doesn't exist.

MCP servers are the only external interface

Agents never call APIs directly.

Plugins are composable

Install/uninstall a department without breaking others.

Vaishak is the override

Any agent decision can be overridden. Agents escalate, never assume.

Automated Workflows

Daily Operations (9:00 AM IST)

[9:00 AM TRIGGER] → Daily Dispatcher reads PROGRESS.md, Pilot Tracker, Finance Ledger, Content Calendar ↓ standup-gen skill → generates Yesterday / Today / Blockers / Metrics ↓ Check for sub-triggers: • New WhatsApp lead? → route to Sales Pipeline • Pilot check-in due? → route to Deal Closer • Content posting day? → route to Social Scheduler • Compliance deadline <7d? → alert Compliance Bot ↓ Output: Standup summary + prioritized task list → Vaishak

Sales Pipeline (Event-Driven)

[NEW LEAD] → whatsapp-mcp reads message → Deal Closer activated ↓ lead-qualify skill → Score 1-10 ↓ Score ≥ 6: proposal-draft → send via whatsapp-mcp → update Pilot Tracker Score < 6: Add to nurture list, monthly check ↓ If pilot agreed: Day 1 deploy → Day 3 check → Day 7 review → Day 14 convert/drop ↓ CONVERT → trigger Client Onboarding Workflow DROP → log reason, nurture list

Content Calendar (Monday 10:00 AM)

[MONDAY TRIGGER] → Social Scheduler reviews calendar + last week's engagement ↓ Content Engine drafts 2 LinkedIn + 2 Instagram pieces ↓ Brand Guardian reviews each (score ≥ 7 to pass) ↓ Queue: Tue 9:30 LinkedIn | Wed 6:00 Instagram | Thu 9:30 LinkedIn | Sat 11:00 Instagram ↓ Vaishak reviews before publish

Weekly Business Review (Friday 5:00 PM)

[FRIDAY TRIGGER] → Weekly Reviewer gathers data from all MCP servers + Knowledge ↓ weekly-review skill → generates KPIs, wins, misses, next week priorities ↓ strategy-check → compare actual vs roadmap, flag off-track items ↓ Output: WBR document → save to GDrive → notify Vaishak

Client Onboarding (Payment Received)

[PAYMENT TRIGGER] → Revenue Tracker confirms via razorpay-mcp → Finance Ledger updated ↓ Client Success → firebase-mcp creates user + gdrive-mcp creates folder ↓ whatsapp-mcp sends welcome template → Installation scheduled ↓ VAISHAK ON-SITE: Mount cameras → Connect Pi → WiFi → Calibrate → Train staff ↓ Post-install: Day 1 verify → Day 3 call → Day 7 data review → Day 14 assessment → Day 30 NPS
