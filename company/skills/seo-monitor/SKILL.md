## On-Page SEO Checklist (Per Page)

Use this for all website pages: homepage, product pages, blog posts, pricing, about, contact.

### Pre-Publishing Checklist

#### Title Tag & Meta Description
- [ ] Title tag is 50-60 characters (max 60 before Google cuts off)
- [ ] Primary keyword in title (within first 3 words for maximum impact)
- [ ] Title includes brand name OR value proposition, not both
  - ✓ "AI POS Bangalore | ShopSense"
  - ✗ "ShopSense – Innovative AI-Powered Edge Computing Platform"
- [ ] Meta description is exactly 155-160 characters (test in Google preview tool)
- [ ] Meta description includes primary keyword naturally
- [ ] Meta description has CTA or value proposition ("Learn how...", "Try for free", etc.)
- [ ] Meta description doesn't repeat title

#### URL Structure
- [ ] URL is lowercase, hyphenated (not underscores)
- [ ] URL includes primary keyword
- [ ] URL is short (under 75 characters)
- [ ] URL is human-readable (not "/p=12345")
- [ ] No URL parameters (use Google Search Console to manage if necessary)
- [ ] If blog post, follows structure: /blog/[topic-keyword]
  - ✓ /blog/offline-pos-system-bangalore-billing
  - ✗ /blog-posts/page1 or /articles/a1b2c3

#### H1 Tag (Page Title)
- [ ] Exactly ONE H1 per page (no multiple H1s)
- [ ] H1 includes primary keyword naturally (position 1-3 words preferred)
- [ ] H1 is descriptive and compelling (not just the keyword)
- [ ] H1 is different from title tag (don't duplicate)
  - Title Tag: "AI POS Bangalore | ShopSense"
  - H1: "AI POS System Built for Bangalore Retail"

#### H2 & H3 Subheadings
- [ ] Page has 3-5 H2 sections (minimum 3)
- [ ] Each H2 includes related keyword or semantic variant
- [ ] H2s are descriptive and user-beneficial (not keyword-stuffed)
- [ ] H3s exist under H2s for complex sections
- [ ] Heading hierarchy is logical (no H3 before H2, no H4 without H3)
- [ ] Heading structure matches page outline

#### Content & Keywords
- [ ] Content includes primary keyword at least 2 times (1st paragraph + 1x elsewhere)
- [ ] Keyword density is 0.5-1.5% (not keyword-stuffed)
- [ ] Content includes semantic keywords (related terms, synonyms)
  - Primary: "AI POS Bangalore"
  - Semantic: "billing system Bangalore", "POS software", "retail billing"
- [ ] Content is 300+ words minimum for SEO value
  - Blog posts: 2000+ words
  - Product pages: 800+ words
  - Service pages: 600+ words
  - Pillar pages (Hub content): 3000+ words
- [ ] Content is written for humans, not search engines (read it aloud)
- [ ] No keyword stuffing (repeated phrases, awkward sentences)
- [ ] Content has structure (intro, body sections, conclusion)

#### Internal Linking
- [ ] 3-5 internal links per page (minimum 3)
- [ ] Links use descriptive anchor text (not "click here")
  - ✓ "Learn how offline billing saves ₹2000/month"
  - ✗ "Read more"
- [ ] Links point to relevant, higher-authority pages
- [ ] Links use complete URLs or absolute paths (not relative)
- [ ] No broken links (test before publishing)
- [ ] Links open in same tab (not new tab, unless very necessary)

#### External Links
- [ ] External links are to high-authority sources
- [ ] External links open in new tab (target="_blank")
- [ ] No "nofollow" unless it's a sponsored link or untrusted source
- [ ] External links are relevant to content

#### Images & Alt Text
- [ ] Every image has descriptive alt text (not empty, not filename)
- [ ] Alt text is 8-12 words (not too short, not a paragraph)
- [ ] Alt text includes keyword naturally (if relevant to image)
- [ ] Image filename is descriptive, hyphenated (not image1.jpg)
  - ✓ offline-billing-system-bangalore-chai-shop.jpg
  - ✗ IMG_12345.jpg
- [ ] Images are optimized for web (<200KB)
- [ ] Images have descriptive captions (if appropriate)
- [ ] Images use proper format (JPEG for photos, PNG for graphics)

#### Meta Tags & Structured Data
- [ ] Meta robots tag is correct ("index, follow" for public pages)
- [ ] Canonical tag is correct (self-referential or pointing to primary page)
- [ ] Open Graph tags are set (og:title, og:description, og:image)
  - For social sharing optimization
- [ ] Schema markup is implemented
  - Product page: Product schema (name, price, rating)
  - Blog post: Article schema (headline, author, datePublished)
  - Homepage: Organization schema (name, logo, location)
  - Local business: LocalBusiness schema (address, phone, hours)

#### Mobile & Accessibility
- [ ] Page is mobile-friendly (test in Google Mobile-Friendly Test)
- [ ] Text is readable on mobile (min 14px)
- [ ] Links/buttons are tap-friendly (min 44x44px)
- [ ] Images scale properly on mobile
- [ ] Forms are mobile-optimized (single column, large inputs)
- [ ] No interstitials blocking content (pop-ups)
- [ ] Headings are logically structured for screen readers
- [ ] Images have alt text for screen readers
- [ ] Links have descriptive text (not "click here")
- [ ] Color contrast meets WCAG AA standards (4.5:1 for text)

#### Page Speed (Core Web Vitals)
- [ ] Largest Contentful Paint (LCP): < 2.5 seconds
- [ ] First Input Delay (FID): < 100 milliseconds
- [ ] Cumulative Layout Shift (CLS): < 0.1
- [ ] Test with Google PageSpeed Insights
- [ ] Check mobile and desktop separately

#### Engagement Signals
- [ ] Page has clear CTA (button, link, form)
- [ ] CTA is visible above the fold (mobile)
- [ ] No auto-playing videos or audio (hurt UX)
- [ ] Page content matches search intent (user searches X, finds X, not Y)
- [ ] Page loads fast (test with PageSpeed Insights)

## Technical SEO Audit Criteria

### Site Architecture
- [ ] Homepage is the most linked-to page (central hub)
- [ ] Site structure is logical (categories/subcategories clear)
- [ ] Navigation is consistent across all pages
- [ ] Breadcrumbs exist (helps crawling and UX)
- [ ] Site has clear hierarchy (pillar pages → cluster content)
- [ ] No orphaned pages (every page linked to from navigation or other pages)
- [ ] Site is 2-3 clicks from any page (depth matters for crawlability)

### Sitemap & Robots.txt
- [ ] XML sitemap exists and includes all important pages
- [ ] Sitemap is submitted to Google Search Console
- [ ] Sitemap is updated automatically (regenerated weekly minimum)
- [ ] robots.txt exists and is not blocking important pages
- [ ] robots.txt allows Google to crawl CSS/JS (needed for rendering)
- [ ] No important pages are marked with "noindex"

### URL Structure
- [ ] URLs are lowercase and hyphenated (consistent)
- [ ] URLs don't use parameters (?,&) for main content
- [ ] URL structure mirrors site navigation (logical)
- [ ] No duplicate URLs (same content, different URLs)
- [ ] Redirects (301) are used for moved pages (not 302)
- [ ] Redirect chains are minimal (URL A → B → C is bad, use A → C)

### Mobile Optimization
- [ ] Site is mobile-first responsive (CSS media queries)
- [ ] Mobile version is equivalent to desktop (not stripped-down)
- [ ] Viewport meta tag is set correctly
- [ ] Fonts are readable on mobile (min 14px, 16px ideal)
- [ ] Links are tap-friendly (44x44px minimum)
- [ ] No mobile-specific interstitials (pop-ups blocking content)
- [ ] Test with Google Mobile-Friendly Test

### Core Web Vitals
- [ ] LCP < 2.5 seconds (largest contentful paint)
- [ ] FID < 100ms (first input delay) OR INP < 200ms (interaction to next paint)
- [ ] CLS < 0.1 (cumulative layout shift, no jumping elements)
- [ ] Monitor monthly with Google Search Console
- [ ] Improve if any metric is in "poor" (red) range

### Structured Data / Schema Markup
- [ ] Homepage: Organization schema (name, logo, contact, location)
- [ ] Product pages: Product schema (name, price, rating, availability)
- [ ] Blog posts: Article schema (headline, author, datePublished, image)
- [ ] Local pages: LocalBusiness schema (address, phone, hours, Google Maps embed)
- [ ] Rich snippets appear in search results
- [ ] Test with Google Structured Data Testing Tool
- [ ] No schema errors in Google Search Console

### SSL & Security
- [ ] Site uses HTTPS (green lock icon)
- [ ] All pages force HTTPS (redirects HTTP → HTTPS)
- [ ] SSL certificate is valid and not expired
- [ ] No security warnings in browser
- [ ] No mixed content (HTTP images/scripts on HTTPS page)

### JavaScript & CSS
- [ ] Google can crawl and render all content (test in Search Console > URL Inspection > View Crawled Page)
- [ ] CSS is loaded from linked files, not inline huge blocks
- [ ] JavaScript is asynchronous (defer or async attributes)
- [ ] No render-blocking resources (files that delay page load)
- [ ] Critical CSS is inlined (above-the-fold styles)

### Images
- [ ] Images are optimized (compressed, right size)
- [ ] Images use next-gen formats (WebP with fallback)
- [ ] All images have alt text
- [ ] Image dimensions are specified in HTML (prevents layout shift)
- [ ] Lazy loading is enabled (images load as user scrolls)

### Canonicalization
- [ ] Canonical tags are correct on all pages
- [ ] Canonical tag points to preferred URL (if duplicates exist)
- [ ] No canonical tag conflicts (page A says it's B, but B says it's A)
- [ ] HTTPS version is canonical (not HTTP)
- [ ] WWW vs non-WWW is consistent

### 404 & Errors
- [ ] 404 page is custom and helpful (not blank)
- [ ] 404 page includes navigation/search
- [ ] Broken internal links are fixed (test with tool like Screaming Frog)
- [ ] Broken external links are removed or fixed
- [ ] No 5xx errors (server errors)
- [ ] No redirect loops (URL A → B → A)

## Link Building Strategy for Bangalore Local SEO

### Tier 1: High-Authority Local Links
- **Google My Business**: Optimize and complete all fields
- **Bangalore startup directories**: Crunchbase, AngelList India, Startup India, YourStory
- **Local tech publications**: YourStory (free mentions), TechCircle (pitch coverage)
- **Bangalore retail associations**: Chamber of Commerce, Retail Association, small business groups
- **University links**: IIT Bangalore, IISC, Christ University (alumni networks, research connections)

### Tier 2: Niche & Industry-Relevant Links
- **Retail tech blogs/news**: TechCrunch India, Khatabook blog, other fintech/retail blogs
- **Small business resources**: MSME networks, local business journals
- **Founder/CEO interviews**: Podcast guest appearances, founder networks
- **Industry reports**: Mentioned in reports about retail tech in India
- **University partnerships**: Research collaborations, case studies

### Tier 3: Content-Driven Links
- **Guest blogging**: Write posts for retail, SMB, fintech blogs (link back to oorulogix.com)
- **Quoted in articles**: Pitch yourself as an expert source for journalists
- **Broken link building**: Find broken links on retail/tech sites, offer your content as replacement
- **Resource pages**: Get listed on "Best POS Systems" or "Retail Tech Tools" roundups
- **Partner links**: Link exchanges with complementary services (accounting, payment, supply chain)

### Outreach Strategy
1. **Identify**: Use Google Search for "Bangalore startups", "retail tech India", "POS systems India", etc.
2. **List**: Create spreadsheet with URL, contact email, outreach angle
3. **Pitch**: Personalized pitch (not mass email)
   - Angle 1: "You covered [competitor]. We have a Bangalore angle you might be interested in."
   - Angle 2: "We're mentioned in this [existing article about retail]. Thought you'd like to know."
   - Angle 3: "Our founder is available for expert quotes on [topic relevant to publication]."
4. **Follow-up**: If no response in 5 days, one follow-up, then move on
5. **Track**: Log in spreadsheet (who linked, when, authority, traffic)

### Link Audit (Monthly)
- Check Google Search Console for referring domains
- Identify spammy/low-authority links (mark for disavow if harmful)
- Monitor backlink growth (target: 5-10 new quality links per month)
- Track anchor text distribution (should be varied, not all keyword)
- Monitor Domain Authority (Semrush/Ahrefs)
